# STC Brand QA Checklist

Use this checklist before approving any page or prompt-driven output.

## 1) Brand Consistency
- [ ] Uses approved logo lockups correctly (header horizontal, footer stacked)
- [ ] Preserves logo clear space and aspect ratio
- [ ] Uses approved color and semantic token system
- [ ] Uses Tahoma as body/UI font
- [ ] Uses one approved heading pairing

## 2) Navigation + IA
- [ ] Top nav matches required model
- [ ] Programs, Insurance, Contact are one click away
- [ ] Low-intent links moved to footer/resources
- [ ] CTA visible in header and final section

## 3) Conversion Quality
- [ ] Primary CTA is obvious above the fold
- [ ] CTAs use approved labels
- [ ] Trust signals are visible early
- [ ] Insurance/access pathway is clear

## 4) UX + Readability
- [ ] Copy is scannable (short sections, clear labels)
- [ ] Spacing rhythm is consistent
- [ ] Form controls and interactive elements are clear
- [ ] Desktop layout is single-flow and non-cluttered

## 5) Motion + Accessibility
- [ ] Motion is meaningful and supports hierarchy
- [ ] `prefers-reduced-motion` behavior is handled
- [ ] WCAG AA contrast passes for text + controls
- [ ] Focus states are visible for keyboard users
- [ ] Touch/click targets are appropriate size

## 6) Technical + SEO sanity
- [ ] Correct page title/description intent
- [ ] Structured heading hierarchy (H1-H2-H3)
- [ ] Image alt text present where needed
- [ ] No broken primary navigation links

## 7) Final Sign-off
- [ ] Brand Lead approval
- [ ] UX approval
- [ ] Content approval
- [ ] Ready for publish/deploy
